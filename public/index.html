SCSS

Smart Campus Safety System.

Bridging the gap between automated physical security and human reporting. A centralized API platform integrating ESP32 real-time monitoring with institutional incident management.

// Core Capabilities

Security Dashboard

IoT Sensor Logging

Live environmental tracking featuring DHT11 Temperature & Humidity data, paired with ultrasonic sensors to determine precise OPEN/CLOSED status of critical campus fire exits.

Visual Incident Reports

Empowering users to submit detailed hazards directly to the database. Features secure image uploads routed through multer middleware for accurate photographic evidence.

Direct JWT Messaging

A secure, built-in chat interface facilitating immediate, two-way communication between students and the centralized security team, fully protected by JSON Web Tokens.

Automated Firmware Alarms

The server dynamically responds to ESP32 POST requests with actionable commands (e.g., sound_alarm: true), instantly triggering local firmware buzzers during breaches.

// Under The Hood

System Architecture

NODE_01

ESP32 Firmware

C++ firmware processing local ultrasonic and DHT11 data, posting to the central API.

  • x-api-key auth
  • Buzzer toggles
  • WiFi payload TX
NODE_02

Node.js API

An Express.js backend handling routing, authentication, and file processing.

  • Controllers
  • JWT Verification
  • Multer Image IO
NODE_03

MySQL Database

Relational storage managed via connection pooling for rapid read/write operations.

  • Password hashing
  • Cron data pruning
  • Chat history logs
